# 本地大模型沉浸式翻译 Chrome 扩展

当前版本：0.2.12

## 新功能

- 支持本机接口：`http://127.0.0.1:8080/v1`
- 支持你的远程 FRP 映射接口：`http://frp4.ccszxc.site:14688/v1`
- 支持两个页面翻译入口：“正文翻译”适合干净阅读文章，“沉浸翻译”会扫描全页面可见英文，接近沉浸式翻译效果。
- 0.2.8 优化沉浸翻译速度：默认每批 6 段、2 路并发、真正多段合并请求、可视区域优先、页面内译文缓存。
- 0.2.8 优化排版：默认“紧凑双语”，短 UI 文本采用轻量同行小译文，正文长段保持块状译文；可在选项页切换“紧凑双语 / 清晰双语 / 仅显示译文”。
- 0.2.9 进一步提高本机吞吐：默认每批 10 段、3 路并发，设置页最高可调到每批 16 段、4 路并发。
- 0.2.9 新增默认开启的划词/划句临时翻译：选中英文单词弹出词义，选中句子或段落弹出整段译文；取消选择、点击别处或按 Esc 后自动消失。
- 0.2.10 修复新打开页面划词/划句翻译未自动启动的问题；现在普通 http/https 页面会自动注入监听，不必先点击插件。
- 0.2.10 修复取消选择后词义/句译浮窗残留的问题，并在页面/选区/悬停翻译结束后 best-effort 释放 llama.cpp 上下文，保持后续翻译高效。
- 0.2.11 加强真实页面自动注入：扩展安装/重载、Chrome 启动、标签页切换、URL 更新和页面加载完成时都会尝试注入，适配 Inoreader 这类单页应用中新文章切换场景。
- 0.2.11 加强浮窗消失逻辑：点击页面其他区域时立即取消旧选区翻译，避免词义/句译残留。
- 0.2.12 修复划词/划句浮窗创建时缺少固定 id 的真实根因：现在浮窗能被可靠删除，不会叠加多个残留浮窗。
- 0.2.12 增加注入前探测，避免 manifest 自动注入和后台主动注入重复执行，降低重复请求和上下文占用。
- 0.2.12 将 llama.cpp 上下文释放改为空闲后延迟执行：页面/选区翻译完成后再集中清理，避免每个小翻译都立即清空导致变慢或竞态乱码。
- 默认支持 `Ctrl + 鼠标悬停段落` 翻译单段，无需每次手动开启；弹窗只保留提示。
- 支持文档翻译页：PDF、DOCX、TXT，并默认生成左原文全文、右译文全文的中英文对照阅读版。
- 文档翻译使用医学文献优化模式：默认 2200 字符大块、4096 输出 token、医学术语保留 prompt，减少碎片化请求。
- 文档翻译先显示解析摘要，并提供“重新翻译失败段”按钮。
- 修复医学文献引用编号如 `[1]`、`[1,2]` 被误判为 JSON 导致单段失败的问题。
- 修复译文片段以逗号、顿号等结尾时被误判为“不完整译文”的问题。
- “每段最大字符”上限 3000 已与内部实际分段逻辑一致。
- “重新翻译失败段”前后会尝试清空模型上下文，减少连续失败。
- 远程部署时可以直接粘贴 llama.cpp Web 地址，例如 `http://frp4.ccszxc.site:14688/#`，插件会自动规范为 API 地址 `http://frp4.ccszxc.site:14688/v1`。
- “测试连接”如果遇到 `/v1/models` 被代理返回 502，会自动改用一次极短聊天请求验证 `/v1/chat/completions`，避免 Web 可用但测试误报失败。
- 如果远程 Web 能用但 `/v1` 系列接口仍返回 502，插件会继续 fallback 到 llama.cpp 原生 `/completion` 接口；测试连接和实际翻译都会走这个兜底。
- 支持清除页面译文。
- 支持调用 llama.cpp `/slots/0?action=erase` 清空模型上下文。
- 支持设置重试次数、超时时间、单段切分长度，降低无效等待。

## 安装 / 更新

1. 打开 Chrome：`chrome://extensions`
2. 开启“开发者模式”
3. 如果已安装旧版，点击扩展卡片上的刷新按钮；如果没有安装，点击“加载已解压的扩展程序”
4. 选择目录：`outputs/local-llm-translator-extension`
5. 打开扩展“选项”，选择接口模式：
   - 本机：`127.0.0.1`
   - 远程 FRP：`frp4.ccszxc.site:14688`

远程电脑上如果你看到 Web 页面 `http://frp4.ccszxc.site:14688/#` 能正常聊天，但插件“测试连接”曾提示 HTTP 502，请更新到 0.2.5 后重新加载扩展。新版会自动把 Web 地址转成 `/v1` API 地址；如果 `/v1` 仍不可用，会再尝试 llama.cpp 原生 `/completion`。

## 推荐 llama-server 启动参数

你当前 Qwen3.6-35B-A3B-MTP 体验好，可以继续用：

```cmd
llama-server.exe -m "C:\Users\zengxiaofeng\llama.cpp\models\gemma.gguf" --host 0.0.0.0 --port 8080 --api-key <你的key> --jinja --reasoning off --reasoning-budget 0 --spec-type draft-mtp --spec-draft-n-max 2 -ngl 99 -c 32768 -b 512 -ub 256 -t 6 -tb 12 -fa on -ctk q8_0 -ctv q8_0 -np 1 --slot-save-path "C:\Users\zengxiaofeng\llama.cpp\slot-cache"
```

`--slot-save-path` 用于启用清空上下文接口。远程访问时请务必保留 API Key，不要裸奔暴露模型服务。

## 使用

### 正文翻译

点击扩展图标 → “正文翻译”。译文会插入到原文段落下面；如果页面已经有正文译文，再次点击会清除译文。弹窗默认保留常用入口：正文翻译、沉浸翻译、文档翻译、清除本页译文、设置/测试连接。

正文翻译会优先抓取 `p/li/blockquote/标题/表格单元格`，也会兼容常见的 `article/main/section/[role="main"]` 下 `div` 形式正文，适合 Inoreader、新闻站和很多现代网页。

### 沉浸翻译

点击扩展图标 → “沉浸翻译”。插件会扫描当前页面所有可见英文文本，包括导航、按钮、标题、新闻列表和正文；适合 Inoreader 这种页面内很多英文 UI/标题都需要翻译的场景。

沉浸翻译会先翻译当前可视区域，再后台补齐页面下方内容。默认 2 路并发适合本机 llama.cpp；如果远程 FRP 延迟较高或模型不稳定，可在选项页把“并发请求数”调成 1。

### Ctrl 悬停翻译

默认已开启。按住 Ctrl，把鼠标移到段落上即可只翻译该段；如果鼠标已经停在段落上，再按 Ctrl 也会触发。

### 划词 / 划句翻译

默认已开启。鼠标选中一个英文单词，会在选区附近弹出单词含义，医学相关词会优先给出医学语境解释。选中一句或一段英文，会弹出临时整段译文。该浮窗不插入正文、不改变页面排版；取消选择、点击页面其他位置或按 Esc 后自动消失。

如不需要，可在扩展选项页关闭“默认开启划词/划句临时翻译”。

### 文档翻译

点击扩展图标 → “文档翻译 PDF / Word / TXT”。

- PDF：使用内置 PDF.js 提取文本。扫描件 PDF 需要先 OCR。
- DOCX：支持 Word `.docx`，不支持老式 `.doc`。
- TXT：直接读取。

拖入文档后会先显示解析摘要：文件名、提取字符数、对照段落数、推荐模式和每段最大字符。确认无误后再点“开始翻译”。

文档翻译完成后可以下载 `.bilingual.html` 中英文全文对照文件：左栏为原文全文，右栏为译文全文，适合保存、打印或再次打开阅读。若仍有失败段，可点击“重新翻译失败段”单独重试。

默认“每段最大字符”为 2200，更接近 llama.cpp 网页端拖入 PDF 的大块翻译体验；如显存或上下文压力较大，可降到 1200–1800。

## 安全边界

扩展只允许请求：

- `http://127.0.0.1/*`
- `http://localhost/*`
- `http://frp4.ccszxc.site:14688/*`

不会调用 DeepL、Google、OpenAI 云端接口。
